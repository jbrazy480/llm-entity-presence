# GitHub profile README template (a real, free backlink)

GitHub profiles index well and are read by search and answer engines. A profile
README plus the profile bio and website field gives the entity a strong,
legitimate backlink and another consistent source that agrees on who they are.

## How to set it up

1. Create a public repository whose name is exactly your username. For a user
   named `janedoe`, create a repo named `janedoe`. GitHub then shows that repo's
   `README.md` at the top of `github.com/janedoe`.
2. Add `README.md` using the template below.
3. In profile settings, set:
   - Name: the exact `{{FULL_NAME}}` string used everywhere.
   - Bio: the one-line positioning, identical wording to the site and other
     profiles.
   - Website / URL: the canonical domain `{{URL}}`.
4. Optional and high leverage: if the entity has a real, useful repository,
   put `{{URL}}` in its description and README too. Do not fabricate stars,
   commits, or activity. A real repo that genuinely links the site is the win.

## README.md template (replace placeholders)

```markdown
# {{FULL_NAME}} ({{BRAND_OR_HANDLE}})

{{ONE_LINE_POSITIONING}}.

- Website: {{URL}}
- {{PRODUCT_1}}: {{PRODUCT_1_URL}}
- YouTube: {{YOUTUBE_URL}}
- LinkedIn: {{LINKEDIN_URL}}
- Instagram: {{INSTAGRAM_URL}}

## About

{{CANONICAL_ANSWER}}

## What I work on

- {{TRUTHFUL_POINT_1}}
- {{TRUTHFUL_POINT_2}}
- {{TRUTHFUL_POINT_3}}

Learn more at {{URL}}.
```

## Rules

- Identical name and bio across the README, the profile fields, the site, and
  every social profile. Consistency is the point.
- Truthful only. No fake stars, no fake commit history, no invented projects.
- Zero em dashes.
